# 时间维度上的事实校正系统
## 客观阐述与设计说明文档（工程 / 设计内部版）

### 1. 产品本质定义（保持）
#### 1.1 系统名称（工作定义）

Temporal Fact Correction System（时间维度上的事实校正系统）

#### 1.2 系统研究对象

本系统不以"事件本身"为研究对象，而以：

围绕事件形成的事实判断、叙事结构、共识性认知，
以及这些判断在时间中的被修正过程

作为核心记录单元。

#### 1.3 关键区分点

事件是客观发生的

判断是人类对事件的解释

本系统记录的不是"发生了什么"，而是：

人们当时如何理解它

这种理解后来如何被证伪 / 修正 / 否定

### 2. 与百科 / 档案 / 新闻系统的结构性差异（保持，但去修辞）
#### 2.1 新闻系统

目标：尽快报道新事实

时间结构：现在 → 过去

错误处理方式：更正 / 澄清（非核心流程）

#### 2.2 百科系统

目标：给出当前公认结论

时间结构：结果导向

错误处理方式：版本覆盖

#### 2.3 档案系统

目标：保存原始材料

时间结构：静态

错误处理方式：并置，不裁决

#### 2.4 本系统（严格定义）

目标：记录"判断如何失败"

时间结构：

判断形成 → 判断被挑战 → 判断被修正 → 判断结案

错误不是异常，而是核心数据

### 3. 核心设计立场（保持）
#### 3.1 系统不追求"信息完备性"

不试图覆盖全部事件

不试图穷尽所有信息源

#### 3.2 系统只关注一种高价值对象

曾形成公共认知、后来被世界修正的判断

这是有限集合，也是系统可持续运行的前提。

#### 3.3 系统对抗的不是"信息不足"，而是：

错误叙事长期存活

被推翻的判断没有"结案机制"

人类记忆对"支持过自己立场的内容"具有结构性偏好

### 4. 系统结构设计（核心修改部分）

本系统分为两大层：

A. 用户可见的显性板块（User-facing Layers）

B. 系统级隐性板块（System-level Layers）

二者逻辑严格区分，禁止混用。

#### A. 显性板块（用户可直接访问）

原则：用户看到的是"认知变化的结果"，而不是系统运作细节

##### A1. 非个性化世界快照（World Snapshot）

功能定义：

展示某一时间点，围绕某事件形成的主流判断集合

模块结构：

A1.1 时间锚点（Snapshot Timestamp）

A1.2 主流判断摘要（Consensus Summary）

A1.3 判断成立的当时依据（Then-valid Evidence）

重要补充（来自原第 5 点）：

用户不需要判断真假

此模块只负责"还原当时世界如何看待它"

##### A2. 认知演变时间轴（Belief Evolution Timeline）

功能定义：

展示一个判断如何被逐步动摇、修正或否定

模块结构：

A2.1 初始判断

A2.2 隐含前提

A2.3 挑战性信息出现

A2.4 判断修正 / 放弃节点

##### A3. 过去 30 天，被证明是错的说法（显性纠错板块）

功能定义：

集中展示近期已被修正或证伪的公共判断

模块结构：

A3.1 判断摘要

A3.2 原判断影响范围

A3.3 修正依据

A3.4 当前状态（结案 / 仍存争议）

设计目的：

对抗"错误自然消失"的信息结构

为认知修正提供公共可见性

##### A4. 认知失败档案（Historical Cognitive Failure）

功能定义：

长期保存具有代表性的认知失败案例

模块结构：

A4.1 判断生命周期

A4.2 错误成因分析

A4.3 修正代价（社会 / 决策 / 认知）

#### B. 隐性板块（系统级，不直接暴露）

原则：用户不承担系统责任，系统不向用户转嫁判断负担

##### B1. 判断版本控制系统（Judgement Versioning）

每一次判断都有版本号

新版本必须指向旧版本

禁止"覆盖式更新"

##### B2. 自查纠错机制（Self-correction Engine）

新信息自动回溯影响的旧判断

要求明确指出：

哪些结论被影响

哪些假设失效

##### B3. 不适感触发机制（Cognitive Dissonance Trigger）

来源于你强调的点，嵌入于系统而非用户负担：

用户只需标记：

"这个判断后来让我困惑 / 不舒服"

系统将其作为：

重新审视判断的触发信号

用户不被要求识别真伪

##### B4. 判断结案状态机（Closure State Machine）

未结案

部分修正

已证伪

条件性成立

这是系统区别于资讯产品的关键。

### 6. 我们真正对抗的问题（保持）

本系统假设：

信息从来不稀缺

更新速度从来不是核心问题

核心问题是：

错误判断在互联网上没有被系统性记忆，
修正缺乏结构化呈现，
导致人类不断重复相同的认知失败。

### 7. 设计哲学与系统伦理（保持，客观表述）
#### 7.1 关于"错误"

错误是常态，不是失败

不记录错误，本身是一种认知不诚实

#### 7.2 关于立场

系统不裁决立场

系统裁决：哪些判断已经失效

#### 7.3 核心原则

世界会不断纠正我们，
本系统的责任，是记住这件事发生过。

### 8. 技术实现的指导性原则（新增，关键）
#### 8.1 禁止目标

❌ 禁止个性化推荐

❌ 禁止以点击率作为主排序依据

❌ 禁止用"最新"覆盖"曾经相信过的"

#### 8.2 必须满足的系统特性

所有判断可回溯

所有修正必须指向历史版本

所有结案状态明确

#### 8.3 Agent / 多 Agent 使用原则（概念层）

Agent 不做裁决者

Agent 只做：

证据收集

判断比对

前后冲突检测

### 结语（给内部）

这不是一个"更聪明的资讯系统"，
而是一个对人类认知失败负责的系统。
