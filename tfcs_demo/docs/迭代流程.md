# 迭代流程规划文档

## 1. 迭代目标

基于最终设计文档和技术背景文档，将当前 Demo 迭代升级为完整的时间维度上的事实校正系统，重点实现：

- 非个性化世界快照
- 认知演变时间轴
- 过去 30 天被证明是错的说法
- 认知失败档案
- 真实 NPU 推理集成

## 2. 迭代阶段划分

### 阶段 1：基础架构优化（Week 1）
- 清理超出强约束的代码
- 完善分层架构
- 集成真实 NPU 推理

### 阶段 2：核心功能实现（Week 2-3）
- 实现判断版本控制系统
- 实现自查纠错机制
- 实现不适感触发机制
- 实现判断结案状态机

### 阶段 3：用户界面升级（Week 4）
- 开发可视化时间轴 UI
- 实现世界快照展示
- 实现认知失败档案查看

### 阶段 4：系统测试与优化（Week 5）
- 性能测试与优化
- 功能测试与修复
- 准备比赛提交材料

## 3. 详细迭代步骤

### 3.1 阶段 1：基础架构优化

#### 任务 1.1：代码清理与架构优化
- 移除 `ai/config.py` 和 `ai/inference/model_factory.py` 等超出强约束的文件
- 简化 `npu_runner.py`，确保返回值严格符合接口契约
- 完善分层架构，确保无跨层调用

#### 任务 1.2：NPU 推理集成
- 基于高通 QNN SDK 实现真实 NPU 推理
- 转换轻量级文本分类模型为 ONNX 格式
- 实现模型加载与推理优化
- 测试 NPU 推理性能与准确性

#### 任务 1.3：基础测试框架搭建
- 编写单元测试验证核心功能
- 实现性能测试脚本
- 建立持续集成流程

### 3.2 阶段 2：核心功能实现

#### 任务 2.1：判断版本控制系统
- 设计判断版本数据结构
- 实现版本管理逻辑
- 确保新版本指向旧版本
- 禁止覆盖式更新

#### 任务 2.2：自查纠错机制
- 实现新信息自动回溯影响的旧判断
- 设计影响分析算法
- 明确指出哪些结论被影响、哪些假设失效

#### 任务 2.3：不适感触发机制
- 设计用户反馈接口
- 实现触发信号处理逻辑
- 集成重新审视判断的流程

#### 任务 2.4：判断结案状态机
- 实现状态定义：未结案、部分修正、已证伪、条件性成立
- 设计状态转换逻辑
- 实现状态持久化

### 3.3 阶段 3：用户界面升级

#### 任务 3.1：可视化时间轴 UI
- 开发基于 Web 的时间轴组件
- 实现交互式时间点选择
- 支持叙事类型演化可视化

#### 任务 3.2：世界快照展示
- 实现时间锚点选择界面
- 开发主流判断摘要展示
- 设计判断依据展示组件

#### 任务 3.3：认知失败档案查看
- 实现档案列表页面
- 开发判断生命周期展示
- 设计错误成因分析与修正代价展示

### 3.4 阶段 4：系统测试与优化

#### 任务 4.1：性能测试与优化
- 测试 NPU 推理性能
- 优化内存使用
- 提升时间轴生成速度

#### 任务 4.2：功能测试与修复
- 测试所有核心功能
- 修复发现的问题
- 验证系统稳定性

#### 任务 4.3：比赛提交材料准备
- 录制 3 分钟演示视频
- 制作技术架构 PPT
- 准备 exe 安装包

## 4. 技术实现要点

### 4.1 NPU 推理优化
- 使用轻量级文本分类模型（如小型 BERT 或 CNN）
- 模型量化为 INT8 以提高 NPU 性能
- 批处理推理以提高吞吐量

### 4.2 异构计算设计
- NPU：负责文本分类推理
- CPU：负责业务逻辑处理
- GPU：负责可视化渲染（可选）

### 4.3 数据管理
- 使用本地文件系统存储数据（符合 Demo 要求）
- 设计高效的数据结构
- 实现数据备份与恢复机制

### 4.4 用户体验
- 简洁明了的界面设计
- 响应式布局
- 清晰的时间轴可视化

## 5. 风险评估与应对策略

### 5.1 技术风险
- **风险**：NPU 推理性能不达预期
- **应对**：准备 CPU 推理作为备用方案，优化模型结构

### 5.2 时间风险
- **风险**：部分功能开发时间不足
- **应对**：优先实现核心功能，确保比赛验收标准

### 5.3 质量风险
- **风险**：系统稳定性问题
- **应对**：加强测试，建立错误监控机制

## 6. 迭代验收标准

### 6.1 阶段 1 验收
- 代码结构清晰，符合分层架构

### 6.2 阶段 2 验收
- 核心功能完整实现
- 系统能正确处理判断版本和状态
- 自查纠错机制有效

### 6.3 阶段 3 验收
- UI 界面美观易用
- 时间轴展示清晰

### 6.4 阶段 4 验收

- 所有功能测试通过
- 比赛提交材料完整
