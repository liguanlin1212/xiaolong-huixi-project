# 实现判断版本控制系统

## 1. 任务分析

根据迭代流程文档，任务 2.1 需要实现判断版本控制系统，具体要求：
- 设计判断版本数据结构
- 实现版本管理逻辑
- 确保新版本指向旧版本
- 禁止覆盖式更新

## 2. 实现方案

### 2.1 扩展核心类型定义

在 `core/types.py` 中添加：
- `JudgementStatus` 枚举：表示判断的状态（未结案、部分修正、已证伪、条件性成立）
- `JudgementVersion` 数据类：包含版本信息、判断内容、状态、创建时间、上一版本引用等

### 2.2 实现版本管理服务

创建 `app/services/version_service.py`：
- `create_new_version()`：基于旧版本创建新版本
- `get_version_history()`：获取判断的完整版本历史
- `get_latest_version()`：获取最新版本
- `validate_version_chain()`：验证版本链的完整性

### 2.3 集成到判断服务

扩展 `app/services/judgement_service.py`：
- 修改 `build_timeline()` 以支持版本化判断
- 添加版本控制相关的辅助函数

### 2.4 编写测试用例

在 `tests/unit/` 中添加：
- `test_version_service.py`：测试版本管理功能
- `test_judgement_version.py`：测试判断版本数据结构

## 3. 技术要点

- 使用不可变数据结构确保版本的不可变性
- 实现版本链以维护历史记录
- 防止直接修改现有版本
- 提供完整的版本历史查询功能

## 4. 实现步骤

1. 扩展核心类型定义
2. 实现版本管理服务
3. 集成到现有判断服务
4. 编写测试用例验证功能
5. 运行测试确保功能正常
