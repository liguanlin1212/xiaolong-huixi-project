# 任务1.1：代码清理与架构优化实施计划

## 目标
根据强约束实现说明文档，执行代码清理与架构优化，确保系统符合分层架构要求。

## 实施步骤

### 1. 移除超出强约束的文件
- **移除 `ai/config.py`**：该文件包含配置管理功能，超出了强约束的范围
- **移除 `ai/inference/model_factory.py`**：该文件包含模型工厂功能，超出了强约束的范围
- **移除 `ai/inference/openai_runner.py`**：强约束只允许 NPU 推理，不允许使用 OpenAI
- **清理相关的 `__pycache__` 文件**：确保代码库干净

### 2. 简化 `npu_runner.py`
- **修改 `classify_text` 函数**：
  - 移除对 config 和 model_factory 的依赖
  - 确保返回值严格符合接口契约（只返回 `label` 和 `confidence`）
  - 简化函数实现，保持核心的关键词规则分类逻辑
- **移除 `NPURunner` 类中的额外字段**：确保返回值只包含必要的字段
- **保持向后兼容性**：确保修改后的函数能被现有代码正常调用

### 3. 完善分层架构
- **检查并修复跨层调用**：确保 UI 层不直接调用 AI 层，AI 层不包含业务语义
- **验证目录结构**：确保目录结构与强约束文档完全一致
- **检查核心数据结构**：确保 `core/enums.py` 和 `core/types.py` 符合要求

### 4. 验证修改结果
- **运行测试**：执行现有的测试脚本，确保系统能正常运行
- **检查代码结构**：确保代码结构清晰，符合分层架构要求
- **验证 AI 接口**：确保 `classify_text` 函数返回值符合接口契约

## 预期结果
- 代码结构清晰，符合强约束文档的要求
- 分层架构完善，无跨层调用
- NPU 推理接口简洁明了，返回值严格符合契约
- 系统能正常运行，通过基础测试
